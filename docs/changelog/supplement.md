梳理的核心修改建议：

### 🚨 明显不足 1：状态转移（Transition）漏掉了最耗时的“螺旋搜索”代价

- **当前计划的漏洞：** 在你的状态转移描述中，航程扣除为 $L_i \leftarrow L_i - d(i, j)$。这仅仅扣除了点到点的**转场直线距离**。

- **与现有代码的脱节：** 你的 `simulator.py` 和 `spiral_search.py` 中，无人机到达目标区域后还需要进行密集的螺旋搜索（`tgt['spiral_dist']`），这部分消耗的航程（或时间）往往比转场距离还要大得多。如果不把这部分扣除，DQN 在推演时会认为自己还有很多航程，导致在物理执行时发生严重的航程超限（坠毁）。

- **修正建议：**

  状态转移中的航程/时间扣除，必须严格对齐 `simulator.py` 中的计算逻辑，包含转场与微观搜索：

  $$L_i \leftarrow L_i - \big( d(i, j) + L_{spiral}(j) \big)$$

### 🚨 明显不足 2：隐式放弃目标的惩罚结算时机

- **当前计划的漏洞：** 奖励函数写的是 $r_t = \alpha \cdot w_j - \beta \cdot d(i, j)$。这个单步奖励只计算了“得到”的收益。但是，如果所有无人机都触发了终止条件，最终导致某些目标（尤其是高权重目标）被遗漏，这部分惩罚在你的新 MDP 中没有体现。

- **修正建议：**

  在新架构下，目标的“放弃”是被动的（即 Episode 结束时依然没有被任何无人机访问的节点）。因此，**必须在回合终止的全局奖励中，加入对未访问目标的严厉惩罚**：

  $$R_{terminal} = R_{success} - \gamma \sum_{k \in \text{Unvisited}} w_k$$

  这样才能倒逼 DQN 在前期的轮询中，优先把高权重目标抢走。

### 🚨 明显不足 3：动作空间掩码与零填充（Zero-Padding）的对齐

- **当前计划的漏洞：** 计划中提到用 Zero-padding 来固定输入状态的维度，但没有明确动作空间如何处理变长目标。DQN 的输出层神经元数量是固定的（即 $N_{max} + 1$）。
- **工程实现难点：** 假设设定最大目标数 $N_{max} = 20$（初始 15 + 预留突发 5）。如果当前环境中只有 16 个目标，你需要将动作 17~20 的输出通过 Masking 强制置为 $-\infty$。同时，那些已经被其他飞机选走的目标、超出当前飞机可用航程的目标，也统统要在输出 Q 值后立即加上 Mask。
- **修正建议：** 在文档中明确定义 **动态动作掩码 (Dynamic Action Masking)** 生成规则：
  1. 索引超出当前实际目标数量的动作 $\rightarrow$ Mask
  2. 目标已被分配 ($M_j = 1$) $\rightarrow$ Mask
  3. $d(i, j) + L_{spiral}(j) > L_i$ $\rightarrow$ Mask
  4. 动作 $0$ (Stop) $\rightarrow$ 永远不 Mask
