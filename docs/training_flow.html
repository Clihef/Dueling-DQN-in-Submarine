03 Reward Design — v2 两段式奖励
中间步骤奖励 (done=False)
compute_step_reward(action, target, routes, pos, consumed_ranges, hotspot_map)

if action == 0 (放弃):
  r_step = −target.weight × 0.5
  (机会成本: 权重越高放弃代价越大)

if action ∈ {1,2,3,4} (指派给UAV_k):
  incremental_dist = ∥UAV_k_last_pos → target∥
  spiral_km = target.spiral_dist / 1000
  extra_km = incremental_dist + spiral_km
  range_after = consumed_ranges[k] + extra_km
  range_pressure = clamp((range_after − 120) / 30, 0, 1)
  // 0 when <80%MAX, linear to 1.0 at 100%MAX
  r_step = −(extra_km / 0.05 / 5000 + range_pressure × 0.5)
  // 物理代价归一化 + 航程压力惩罚

NORM = 5000s (≈150km飞行时间)
终端步骤奖励 (done=True)
dqn_val = Jmax + 0.1 · Jsum

fatal_penalty = range_violations × 0.2 × dqn_val
  // 每架超150km的UAV扣20%dqn_val

abandon_penalty = Σ abandoned[i].weight × 5000

r_term = (baseline_val − dqn_val − fatal_penalty − abandon_penalty)
       / max(baseline_val, 1.0)

→ DQN优于baseline → reward > 0
→ 越限或放弃过多 → heavy penalty → reward < 0
设计思想: 中间步骤提供即时学习信号（增量代价 + 航程压力），保留 MDP 长期规划能力（done=False）。 终端步骤提供全局评价（vs baseline + 越限惩罚），确保 DQN 追求整体最优而非贪心。 两段式奖励避免了将 MDP 退化为 Contextual Bandit 的问题。
